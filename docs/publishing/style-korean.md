# 한국어 문체 가이드: AI 티 제거 (번역투·어색함 방지)

한국어 글이 **어색하거나 "AI가 쓴 티"가 나지 않도록** 하는 규칙. 글을 쓰거나 다듬을 때 반드시 적용한다.
출처: [epoko77-ai/im-not-ai](https://github.com/epoko77-ai/im-not-ai) `humanize-korean` 스킬 (MIT) 의 규칙을 이 블로그에 맞게 정리.

> 영어 글은 [style-english.md](./style-english.md)를 따른다.

## 절대 건드리지 않는 것 (Do-NOT)

고유명사·제품명·모델명·기관명, 수치·날짜·단위, 큰따옴표 안 직접 인용, 코드·식별자, 수학/통계 표기, 업계 표준 영어 약어(LLM·GPU·API·MCP 등). 윤문 변경률이 30%를 넘으면 과윤문이다. 멈추고 재검토한다.

## A. 번역투 (가장 흔한 AI 티)

| 패턴 | 고치기 |
|------|--------|
| "~에 대해(서)" | 목적격으로 직결 ("X에 대해 논의" → "X를 논의") |
| "~를 통해" 남발 | "~로", "~해서", "~함으로써"로 분산 |
| "~에 있어(서)" | "~에서", "~을 볼 때" |
| "~와 관련하여/관련된" | "~에", "~의" |
| "가지고 있다" (have/make/take 직역) | 동사·형용사로 ("경쟁력을 가지고 있다" → "경쟁력이 강하다") |
| 이중 피동 "~되어진다" | 단일 피동 ("판단되어진다" → "판단된다") |
| "~에 의해" 피동 | 행위자를 주어로 ("AI에 의해 생성" → "AI가 만든") |
| "~할 수 있다" 남발 | 단언으로 ("높일 수 있다" → "높인다") |
| 영어 대명사 직역 "그/그것/그들" 반복 | 생략하거나 명사로 |
| 명사 앞 긴 관형구(3어절+) | 문장을 분리 |

## B. 영어 병기 과다

- 한글 옆 괄호 영어를 매번 달지 말 것. **첫 등장만 병기**, 이후엔 한글(또는 표준 약어)만.
- 직역 가능한 영어 어휘는 한국어로. 단 업계 표준 용어는 원어 유지.

## C. 구조적 AI 패턴

- **이모지 남발 금지** (기술 글이면 거의 다 제거)
- "먼저·반면·결국" 3단 공식, "A인가·B인가" 대구 반복 → 1회로 줄이거나 평서문으로
- "X: Y" 콜론 부제 헤딩 반복 금지
- 연결어미(-고/-며/-지만/-아서) 직후 **쉼표 남발 금지**

## D. AI 특유 관용구 (Signature phrases): 삭제 1순위

- 결산 피벗: "결론적으로/따라서/이를 통해/요약하면/정리하면" 3회 초과 금지
- "시사하는 바가 크다 / 주목할 만하다 / 본질적으로 / 핵심적으로" → 삭제 또는 구체 결론으로
- hype 어휘: "파격적·압도적·강력한·획기적·치명적" → 구체 수치·사실로
- 의인화 추상 주어: "기술이 묻는다 / 시대가 부른다" → 사람·기관 주어로
- 결말 공식: "~할 때다 / ~해야 한다 / 지금이야말로" → 평서로 닫기

## E. 대시·헤딩 (AI 특유 형식 패턴)

### 대시 금지 (em dash / en dash)

em 대시(—)와 en 대시(–)는 AI 글의 강한 신호다. 산문에서 쓰지 않는다.

| 상황 | 대체 방법 |
|------|-----------|
| 부연·삽입구 | 괄호 또는 쉼표로 분리 |
| 문장 전환 | 마침표로 끊거나 콜론(:) |
| 목록 항목의 설명 구분 | 콜론 또는 자연스러운 서술 ("Langfuse: 오픈소스 관측 대표 주자") |

헤딩에서도 "제목 — 부제" 형식을 쓰지 않는다. 부제는 괄호 또는 콜론으로.

### 의문문 헤딩 금지

"무슨 문제를 푸나?", "왜 필요한가?", "어떻게 다른가?" 같은 수사 의문문 헤딩은 AI·listicle 글의 전형적인 형식이다. 독자에게 정보를 주지 못하고 SEO에도 도움이 안 된다.

**대신:** 명사구 또는 평서형 선언 헤딩을 쓴다.

| 금지 | 대체 |
|------|------|
| 무슨 문제를 푸나 | 해결하는 문제 |
| 왜 필요한가 | 배경과 동기 |
| 다른 도구와 뭐가 다른가 | 경쟁 도구와 차이 |
| 어떻게 설치하나 | 설치 방법 |

헤딩은 짧고 사실 중심으로. 의문부호로 끝나면 일단 의심한다.

## F. 리듬·종결어미

- 문장 길이를 의도적으로 들쭉날쭉하게 (단문/장문 섞기). 균일 = AI 티.
- "~다" 4문장 연속 금지. "~었다·~ㄴ다·~는다·~ㄹ 것이다" 등으로 다양화.
- "~고 있다"는 단순 시제로 환원 가능하면 환원 ("읽고 있다" → "읽는다").

## G. 그 외

- "~적 N" 추상 체인("전략적 함의") → 풀어쓰기. 한자어/영어 명사화 누적 자제.
- Hedging("~것이다 / ~로 보인다 / 양쪽 모두") 남발 → 단언 가능한 곳은 단언.
- 문두 접속사("또한·따라서·즉·게다가") 5회+ → 대량 제거. 문장 자체로 흐름 잡기.
- "~인 것이다 / ~라는 점에 있다 / ~다는 의미다" 형식명사 결말 → 평서·직설로.
- 마크다운 ** 강조, 따옴표 강조 남발 금지. 핵심 한두 개만.

## 자체 점검 (글 쓴 뒤 5초)

1. 고유명사·수치·날짜·인용을 한 글자도 안 바꿨나
2. 변경/과장이 과하지 않나 (원 의미 보존)
3. 격식 일관 (평어/경어 혼용 금지)
4. 위 A~D의 S1(고심각) 패턴이 남아 있지 않나
